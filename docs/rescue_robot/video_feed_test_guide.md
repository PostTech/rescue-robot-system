# 실시간 로봇 비디오 스트리밍 테스트 가이드 (Live Video Streaming Guide)

본 가이드는 재난 수색 로봇의 실시간 카메라 영상(열화상/RGB) 프레임을 백엔드로 전송하고, 이를 웹소켓(WebSocket) 스트림을 통해 관제 대시보드 웹 UI에 즉시 시각화해 뿌려주는 테스트 환경을 구성 및 구동하는 방법에 대해 상세히 설명합니다.

---

## 1. 실시간 비디오 피드 아키텍처 개요

본 테스트 환경은 실제 하드웨어 로봇의 비디오 드라이버나 대역폭 제어 로직이 없는 환경에서도, 영상 전송과 관제 화면 렌더링 과정을 모던 웹 표준 기술로 무결하게 모사할 수 있도록 유기적으로 설계되었습니다.

```text
┌──────────────────────────┐             ┌────────────────────────┐
│  send_test_frame.py      │             │  FastAPI Web Server    │
│  (Thermal Frame Gen)    │ ───────────> │  (:8000/api/demo/      │
│  Base64 Jpeg, 10 FPS     │    [POST]   │   send-frame)          │
└──────────────────────────┘             └────────────────────────┘
                                                     │
                                                     │ [WebSocket Broadcast]
                                                     ▼
┌──────────────────────────┐             ┌────────────────────────┐
│  index.html (Dashboard)  │ <─────────── │  WebSocket.js Client   │
│  (Render <img> base64)   │  [Reactive]  │  (Event: video.frame)  │
└──────────────────────────┘             └────────────────────────┘
```

1.  **프레임 시뮬레이터 (`send_test_frame.py`)**: 
    *   파이썬 Pillow(PIL) 라이브러리를 사용하여 $320 \times 240$ 크기의 실감 나는 FLIR 열화상 프레임 데이터를 초당 10프레임(10 FPS) 속도로 동적 생성합니다.
    *   시간에 따라 움직이는 인명 열원 실루엣, 3D Scan Line 그리드 노이즈, 실시간 녹색 AI 바운딩 박스(신뢰도 % 연동 표시)를 실시간으로 그리고 JPEG 바이너리로 변환하여 Base64로 인코딩합니다.
    *   인코딩된 프레임을 FastAPI `/api/demo/send-frame` API로 초당 10번 전송(POST)합니다.
2.  **FastAPI 라우터 및 웹소켓 브로드캐스터**:
    *   서버는 전송받은 base64 이미지 데이터를 가로채, 현재 접속된 모든 웹소켓 클라이언트 세션에 `video.frame` 이벤트 페이로드로 즉시 지연 없이 브로드캐스트(Broadcast)합니다.
3.  **웹 브라우저 UI 렌더링 (`websocket.js` & `index.html`)**:
    *   웹소켓 클라이언트는 `video.frame` 이벤트를 실시간 가로채, 디코딩 단계 없이 브라우저 기본 이미지 성능을 활용할 수 있도록 `<img id="live-camera-feed">`의 `src="data:image/jpeg;base64,..."` 속성에 바인딩시킵니다.
    *   대기 화면(`camera-placeholder`)이 스위칭되며 부드러운 로봇 카메라 영상 피드가 화면 우측 패널에 가시화됩니다.

## 2. 스트리밍 테스트 환경 구동 순서

개발 폴더(`c:\Users\cosmo\AI_challange`)에서 아래 순서대로 구동하여 실시간 카메라 뷰를 체험할 수 있습니다.

### 1단계: 통합 데모 웹 서버 실행
새로운 터미널(PowerShell 또는 CMD)을 열고 아래 명령어를 입력하여 통합 서버를 가동합니다.
```powershell
cd "c:\Users\cosmo\AI_challange"
python run_mvp.py
```
> [!NOTE]
> 서버 실행 시, 백엔드 라우터와 프론트엔드 대시보드가 `8000`번 단일 포트에서 통합 서빙됩니다.

### 2단계: 웹 대시보드 브라우저 접속
크롬(Chrome) 등의 브라우저를 켜고 아래 주소로 접속합니다.
*   **주소**: **[http://localhost:8000](http://localhost:8000)**
*   접속 직후 우측 상단에 **[로봇 실시간 카메라 피드 (Live Feed)]** 패널이 신설되어 **"실시간 비디오 스트림 대기 중..."** 이라는 애니메이션 메시지와 함께 스트리밍 대기 상태에 들어간 것을 볼 수 있습니다.

### 3단계: 비디오 프레임 스트리머 가동
서버가 켜져 있는 상태에서, **새로운 터미널(쉘) 창을 하나 더 열고** 다음 명령을 실행해 비디오 전송 스크립트를 가동합니다.
```powershell
cd "c:\Users\cosmo\AI_challange"
python send_test_frame.py
```
> [!TIP]
> **의존성 자동 설치**: 만약 사용자의 파이썬 가상환경에 이미지 생성용 Pillow 라이브러리가 없더라도, 본 스크립트 실행 즉시 내부 자가치유 코드에 의해 `pip install Pillow`를 백그라운드에서 자동 가동하여 설치 및 이행을 마무리합니다.

## 3. 웹 화면 시각적 확인 및 인터랙션

1.  **비디오 스트림 로드**:
    *   `send_test_frame.py` 스트리머가 실행되면 브라우저 대시보드의 우측 패널 대기 상자가 자동으로 사라지고, **320x240 크기의 모던 열화상 비디오 뷰**가 나타납니다.
2.  **동적 그래픽 연동 감상**:
    *   중앙 크로스헤어 정밀 조준점과 붉은색 구조 대상자(Human hot spot) 실루엣이 부드럽게 좌우상하로 움직이는 10 FPS 애니메이션이 재생됩니다.
    *   실외/실내 온도 텔레메트리 텍스트와 함께, Sensor Fusion 감지 결과를 모사한 **녹색 AI Bounding Box**가 실종 실루엣의 크기에 맞추어 실시간으로 감싸 돌며 신뢰도 스코어(`VICTIM ALIVE: 95.8%` 등)를 유동적으로 실시간 변경하는 시각적 쾌감을 제공합니다.
3.  **동작 종료**:
    *   테스트 영상 전송을 중단하려면, 스트리머가 돌고 있는 터미널 창에서 **`Ctrl+C`**를 눌러주면 됩니다.
    *   전송이 중단되면 웹 화면은 다시 우아하게 "비디오 스트림 대기 중" 상태로 조화롭게 복귀합니다.
